# astrbot_plugin_wca 协作说明

WCA 与 one 平台成绩查询工具集，包含 WCA 查询、one 查询、成绩图片、PK 对比、跨平台 PR、绑定、宿敌查询、近期比赛和帮助页等功能。

## 初步印象

- 插件入口是 `main.py`，负责初始化各服务、读取配置，并注册 AstrBot 命令。
- 代码已经按职责拆分：`clients/` 放外部 API 客户端，`core/` 放查询、绑定、格式化、反馈等核心能力，`services/` 放面向命令的业务服务。
- `templates/` 用于帮助页和个人成绩卡片渲染，属于用户体验相关资源，改动时要同时关注不同平台的图片/HTML 渲染效果。
- 当前版本 `v1.1.5` 是轻量 API 架构，不拉本地 WCA 数据库；实时性好，但更依赖外部 API 的稳定性。

## 改动原则

1. 新命令优先新增独立 service，再由 `main.py` 初始化和注册，保持入口文件只做装配。
2. 外部 API 调用必须考虑超时、非 200 响应、字段缺失、重名选手、空成绩等边界。
3. WCA 与 one 的项目映射、成绩格式化和排名展示是核心业务规则，改动时要优先保证兼容旧指令输出。
4. 绑定数据应继续放在插件数据目录，不要写入源码目录；涉及数据结构变化时要考虑迁移和回退。
5. 改动命令、配置项、输出规则或版本时，同步更新 `README.md`、`metadata.yaml` 和 `_conf_schema.json`。
